# robots.txt
(아주 가아끔 초보적인 CTF 문제에 등장하는) robots.txt는 사람이 읽는 파일이 아니라 검색엔진에서 색인을 생성하기 위한 봇이 읽는 파일이다.
검색엔진에 봇들은 하이퍼링크를 따라 떠다니면서 다녀간 사이트들의 색인(index)를 생성하는데, **도메인의 최상위 폴더(/)에** robots.txt가 존재하면 해당 파일을 참고해 접근해도 되는 파일들을 파악한다.

이 블로그의 [robots.txt](/robots.txt)는 다음과 같이 구성되어 있다.
``` yaml
User-agent: *
Allow: /

Sitemap: https://pngwna.github.io/sitemap.xml
```
해당 robots.txt는 모든(*) User-agent(주로 봇)가 이 도메인 아래 모든 파일(/)을 탐색해도 된다고 선언한다.
(또한, Sitemap의 위치 역시 표시하는데, sitemap은 후에 살펴보자)

만약 본인이 네이버가 싫어서 네이버의 검색엔진 봇(Yeti)의 검색을 막고자 한다? 그렇다면 다음과 같은 내용을 robots.txt의 앞에 선언하면 된다.
``` yaml
User-agent: Yeti
Disallow: /
```
이 블로그에 위와 같은 내용이 있었다면 네이버의 Yeti는 이 블로그를 탐색하지 않는다.

또는 특정 디렉토리 아래 있는 웹페이지들의 검색만 막고자 하면 다음과 같이 선언하면 된다.
``` yaml
User-agent: Googlebot
Disallow: /admin/ 
```
이렇게 선언한 경우에는 구글봇은 admin 폴더 아래 있는 내용을 탐색하지 않는다.

해당 내용들을 다 합치면 이러한 규칙이 탄생하게 된다. 

``` yaml
User-agent: Yeti # 네이버 검색엔진 봇에 대해서
Disallow: / # 어떠한 탐색도 허용하지 않음

User-agent: Googlebot # 구글 검색엔진 봇에 대해서
Disallow: /admin/ # admin 폴더 아래에 대한 탐색을 허용하지 않음 (이렇게 설정하면 admin 빼고는 다 탐색함)

User-agent: * # 나머지 봇에 대해서
Allow: / # 모든 페이지에 대한 탐색을 허용함

Sitemap: https://pngwna.github.io/sitemap.xml # Sitemap의 주소는 여기 있으니 참고하면 됨
```
이런 방식으로 robots.txt를 잘 수정해서 **최상위 폴더(/)** 아래 넣으면 된다.

Github pages를 사용하는 경우에는 레포지토리 최상위 폴더에 robots.txt라는 이름으로 파일을 넣고 add -> commit -> push 해주면 되고
무료 호스팅을 사용하면 FTP 등을 사용해서 올리면 된다.

# sitemap
sitemap은 해당 웹사이트의 구조를 설명하는 파일이라고 설명하면 쉽게 이해할 수 있을 것이다.
(말 그대로 site의 map이라 더 설명할 방법이 없다...)
보통 XML로 주는데, 이 블로그의 sitemap를 살펴보자.

``` bash
curl https://pngwna.github.io/sitemap.xml
```

sitemap의 index를 도메인 최상위에 올려놨는데, 위와 같이 서버에 request를 날리면 확인할 수 있다.

다음은 response로 들어오는 sitemap.xml이다. 

``` xml

<sitemapindex xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">


    <sitemap>
        <loc>https://pngwna.github.io/post-sitemap.xml</loc>
        <lastmod>2020-03-05T15:00:00.000Z</lastmod>
    </sitemap>

    <sitemap>
        <loc>https://pngwna.github.io/page-sitemap.xml</loc>
        <lastmod>2020-03-05T06:41:00.510Z</lastmod>
    </sitemap>

    <sitemap>
        <loc>https://pngwna.github.io/category-sitemap.xml</loc>
        <lastmod>2020-03-05T15:00:00.000Z</lastmod>
    </sitemap>

    <sitemap>
        <loc>https://pngwna.github.io/tag-sitemap.xml</loc>
        <lastmod>2020-03-05T15:00:00.000Z</lastmod>
    </sitemap>

</sitemapindex>
<!-- XML Sitemap generated by Hexo SEO Friendly Sitemap Generator -->
```

이정도만 알아도 수동으로 sitemap을 작성할 수 있다. 정확한 명세는 [sitemap.org](https://www.sitemaps.org/ko/protocol.html)에서 확인할 수 있다.

수동으로 만들고 싶으면 위와 같은 xml형식을 지켜서 만들면 되지만 보통 자동으로 생성해주는 방법이 존재하기 마련이다.
hexo는 Sitemap Generator를 설치하면 웹사이트가 생성될 때 자동으로 sitemap.xml을 생성해준다.

hexo의 Sitemap Generator를 설치하고자 하면
``` bash
npm install hexo-generator-seo-friendly-sitemap
```
로 설치하고 hexo 폴더의 최상위에 있는 _config.yml에 
``` yaml
sitemap:
  path: sitemap.xml # 원하는 경로
```
를 추가해주면 알아서 생성해준다. 

hexo를 쓰지 않는다면 [자동으로 sitemap을 생성해주는 서비스](https://www.xml-sitemaps.com/)를 사용하거나 수동으로 생성해서 올리자.

# webmaster
robots.txt와 sitemap.xml이 완성되었다. 이제 검색엔진이 우리 사이트를 보다 편하게 탐색할 수 있다!

하지만 수동적으로 검색엔진이 우리 사이트를 찾아서 열심히 탐색해주기를 바라는 것보다 더 빠른 것은 우리가 능동적으로 검색엔진에 우리 사이트를 등록하는 방법이다.

두 가지 방법을 다 사용하면 왠지 검색엔진이 우리 블로그를 더 좋아해줄 것 같다. 둘 다 해보자.

## google 
[Google Search Console](https://search.google.com/search-console/about)에 들어가자.

다음과 같은 화면이 우리를 반겨준다.
![](/images/seo/1.png)
<center> Google Search Console </center>

웹사이트를 구글 서치에 등록하기 위해서는 사이트의 소유권을 증명해야 하는데, 주로 다음과 같은 방식으로 이루어진다.
* DNS Record에 특정 내용 추가 (왼쪽)
* 웹페이지에 특정 HTML 파일 업로드 (오른쪽)
* 웹페이지에 특정 meta 태그 추가 (오른쪽)
* 애널리틱스 연동 (오른쪽)

가비아나 다른 도메인 서비스를 이용해서 도메인을 구매했다면 왼쪽 방법을 사용해도 된다.
하지만 이 블로그는 Github Pages 위에 올라가 있으니 오른쪽 방법중 HTML 파일을 업로드 하는 방식을 사용하도록 하자.
(무료 호스팅의 경우에도 DNS 레코드를 변경할 수 없는 경우가 많으니 해당 방법을 따라하면 된다)

우선 오른쪽에 본인의 웹사이트의 URL을 입력하면 소유권 확인 방법이 나오는데, 첫번째 방법을 사용하기로 했으므로 첫번째 방법에 대해서 알아보자.
![](/images/seo/2.png)
<center> HTML 파일 업로드 권한을 통한 소유권 인증 </center>

특정 파일을 최상위 폴더에 올릴 것을 요구하고 있는데, 해당 html 파일을 다운받아서 올리면 끝난다.

해당 파일에 무슨 내용이 있는지 궁금할 것 같은데, 별거 없다. 

``` json
google-site-verification: $(파일 이름).html
```
인증내용 하나만 들어있는 파일이다.
(참고로 네이버도 거의 똑같은 방식을 소유권 확인에 사용한다)

해당 파일을 업로드하고 확인을 누르면 소유권 확인이 끝나고 서치 콘솔이 열린다.
![](/images/seo/3.png)
<center> Search Console 메인화면 </center>

지금 해줄 것은 sitemap 등록이니 색인 > sitemap에 들어가자

![](/images/seo/4.png)
<center> Sitemap </center>

여기서 자신의 sitemap.xml의 위치를 등록해주면 된다. 
성공하면 아래처럼 '성공'으로 표시가 될 것이고 오류가 있으면 '오류'가 표시가 될 것이다.

나머지 메뉴는 심심할때마다 클릭해보고, 일단 여기까지만 진행해놓자.

## Naver
네이버도 거의 똑같은 방법으로 진행해주면 된다.
[Naver Search Advisor](https://searchadvisor.naver.com)에 들어가서 위에 있는 "웹마스터 도구"를 클릭하면 비슷한 화면이 뜰 것이다.

사이트 추가에 똑같이 본인의 웹페이지를 올린 후 HTML파일을 업로드하면 된다.

![](/images/seo/5.png)
<center> 소유권 인증 </center>

구글과 같은 방법을 사용하고 있는데, HTML 파일의 내용도 같다.
``` json
naver-site-verification: $(파일 이름).html
```

HTML 파일을 올리고 소유권 확인을 끝내면 역시 비슷한 창이 뜨는데, 일단은 요청 > 사이트맵 제출에서 아까와 비슷한 방식으로 사이트맵을 제출하자.

![](/images/seo/6.png)
<center> 서치어드바이저 메인 </center>

이렇게 해주면 등록이 끝났다. 다른 메뉴는 심심할때마다 둘러보자.

# google analytics
앞에서 설명한 webmaster가 잘 검색되는 것을 도와주는 도구라면 analytics는 웹사이트를 이용자, 이용시간, 이용방식등을 분석하는데 도움을 주는 도구다. 
우선 [Google Analytics](https://analytics.google.com)로 들어가자
