# 웹사이트 마련의 꿈
언제나 웹사이트를 만들어보고는 싶었지만 그럴 동기가 없던 참에 {% post_link server/0 "서버도 생기고" %}  도메인도 생기고 최근에 프론트 백을 둘 다 다뤄볼 경험이 있어서 이번에 적기라고 생각했습니다. 그래서 일단 동아리 홈페이지를 먼저 만들어볼까 합니다. 

그런데 같이 할 사람이 없고 해서 혼자 이번학기 내로 만들어볼까 합니다.

이용할 수 있는 인프라는 동아리 선배가 가끔씩 해주는 코드/아키텍처 리뷰밖에 없네요...

# 목표
일단 동아리 웹서비스에서 뭐가 있어야 할지 생각해봅시다.

* 커뮤니티 서비스(게시판 등)
* 계정 권한 관리
* 아카이브
* 공지 기능
* 사진전 갤러리
* 기타 꿀잼 PPAP 기능들

등이 있으면 좋겠네요. 해당 기능들을 제공하는 것을 목표로 합시다.

# 아키텍처
웹서비스를 제공하는데에는 여러 프로그램들의 조합(스택)이 필요합니다. 여러가지 유명한 스택이 있는데, 그 중 MEAN스택은 MongoDB, Express, Angular, Nodejs을 개발 환경으로 선택하여 웹서비스를 만드는 방법입니다. 그런데 저는 MongoDB를 싫어하니 저 DB 대신에 착한 코끼리(PostgreSQL)를 쓰도록 합시다. 그러면 PEAN스택이네요...

해당 환경을 기반으로 대충의 청사진을 그려봅시다.

일단 개발중에 많이 바뀔 것 같지만, 대략적인 구성을 그려봅니다.

![](/images/kuaaa/0.jpg)

Angular쪽이 비어있는 것은 아직 angular를 배운 적이 없어서 그렇습니다.
